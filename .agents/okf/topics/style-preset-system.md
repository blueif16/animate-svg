# Why / how it works (the lifecycle, end to end)
`StylePreset` is the root style overlay wrapper ‚Äî placed OUTSIDE `<SceneFrame>` in a lesson scene, it injects
the active style's shared `<defs>` (e.g. ink-wash's `InkWashDefs`, paper-grain/ink-jitter filters) and publishes
the resolved filter IDs + palette via `StyleContext`. Its `style` prop (a `StyleId`) defaults to `"default"` ‚Äî a
deliberate PASSTHROUGH that emits no DOM and overrides no context, so a lesson that never opts in pays zero
behavior cost. The convention (documented in `research/svg-animation-craft-round2-2026-05-27.md`, per this
file's own header comment) is "sibling-Defs + filter URL" ‚Äî the SAME pattern `FXDefs` uses
([[motion-primitives-library]]) ‚Äî rather than each styled element cloning its own filter defs. Scene code never
imports a style directly; it calls `useStyleFilter()` (or `useStyleFilters()`/`useStylePalette()`) to read
whatever style is currently active off `StyleContext` and applies the returned filter id to its own `<g>`. A
`seed` prop lets a per-instance override reseed procedural style elements deterministically (same seed ‚Üí same
render). `ink-wash` is the one concrete style shipped today (`InkWashBackground`, `InkWashDefs`,
`INK_WASH_FILTER_IDS`, `INK_WASH_PALETTE`, re-exported through `styles/index.ts`).
