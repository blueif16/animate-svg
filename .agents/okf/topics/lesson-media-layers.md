# Why / how it works (the lifecycle, end to end)
`Complete<CamelId>Lesson.tsx` ([[composer-scene-assembly]]'s top-level wrapper) layers THREE parallel media
tracks over the scene, each a thin local file wrapping most of its real logic out to the vendored kits.
`LessonAudioLayer` is genuinely local — it places the per-cue voiceover `<Audio>` spans reading the reconciled
cue windows (from [[timeline-reconcile]]'s `cueAccessors`), the one layer with real in-repo placement logic.
`LessonBgmLayer` is a ONE-LINE re-export (`export { BgmLayer as LessonBgmLayer } from "@studio/sound-kit"`) —
the actual music-bed mixing engine lives entirely in the external `@studio/sound-kit` package, same
sibling-vendored-package boundary [[sound-design-sfx]] and [[timeline-reconcile]] already document for their own
externals; `audioMix.ts` and `mediaSrc.ts` are the SAME shim pattern (`MixConfig`/`NarrationWindow` types and
the `mediaSrc` helper both re-exported straight from `@studio/sound-kit`/`@studio/narration-kit`). What IS local
and load-bearing: `LessonCaptionLayer` renders caption text bound to `LabelWindow`s, and
`filterCaptionsAroundLabels()` is the local logic that suppresses/adjusts a caption when it would overlap a
label-window's on-screen text (a legibility rule, not upstream in either kit); `captionBand.ts`'s `CAPTION_BAND`
is the fixed on-screen bbox every lesson's caption ribbon occupies — the shared footprint
[[render-pipeline]]'s measured pass checks for collisions against, and the "captionBand" zone
[[composer-scene-assembly]]'s `ZoneName` vocabulary declares.
