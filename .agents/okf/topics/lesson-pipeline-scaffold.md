# Why / how it works (the lifecycle, end to end)
`setup-scaffold` is wave 0 of lesson-build: `scaffold-lesson.mjs`'s `main()` validates the one kebab-case
`lessonId`, derives its camelCase/PascalCase forms, and substitutes `lesson-data/_template/pipeline.json` (the
per-lesson pipeline contract — fps/entry/composition/cuePlan/voice out-paths) into a fresh
`lesson-data/<id>/`. `lesson-animatic.mjs` is the animatic previewer later waves (and humans) use to sanity-check
the reconciled cue timeline before a full render exists — it reads the SAME reconciled cues
`w3-5-reconcile` produces via `loadReconciledCues()`, falling back to `runNoSceneFallback()` when no composer
scene exists yet. `remotion.config.ts` sets the process-wide Remotion CLI options (image format, tailwind
webpack override) every render (`w5-render`) inherits.
