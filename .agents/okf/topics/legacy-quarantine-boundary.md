# Why / how it works (the lifecycle, end to end)
`src/lessons/legacy/` holds the PRE-v4 lesson implementations (Scene + Timeline + Complete-wrapper triples for
`ComparisonLesson`, `FenYuHeLesson`, `Kp1FenYuHeIntroLesson`, `Kp1HelloGreetingsLesson`, `MakeTenLesson`,
`PinyinToneLesson`, `TenOnesMakeOneTenLesson`) — kept only so these fixtures stay browsable in Studio, never
touched or extended. `check-legacy-boundary.mjs` (run by `npm run lint`, and the pre-commit gate) is a pure
fs-walk with two rules, both must hold: (a) nothing OUTSIDE `legacy/` may import FROM `legacy/` — walking every
`.ts`/`.tsx`/`.mjs`/`.js` file and flagging an import whose specifier resolves under `LEGACY_DIR`, except the
ONE path in `ALLOWED_LEGACY_BRIDGES` (`src/Composition.tsx`, the Studio barrel that re-exports each legacy
`Complete<X>Lesson` + its duration so they remain registered compositions); (b) no v4 lesson timeline
(`src/lessons/*Timeline.ts`, excluding `legacy/`) may import `reconcileCueTimeline`, `findBoundarySilence`, or a
`*Silences` module — the PRE-v4 silence-reconcile path [[timeline-reconcile]]'s `reconcileClipTimeline` (the
external narration-kit call) superseded. A breach of either rule prints every offender + exits non-zero; a clean
run prints "Legacy-boundary gate OK."
