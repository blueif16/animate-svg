# Why / how it works (the lifecycle, end to end)
`w3b-primitive-build` is the ONLY node allowed to add a new SVG primitive or reusable component. The registry is
CODE-AS-TRUTH: `build-registry.mjs`'s `buildPrimitives()` derives `component`/`kind`/`source`/`motionVocabulary`
straight from `src/shape-primitives/index.ts`'s barrel exports (never hand-edited — `primitive-registry.json`'s
own header comment says so), validated against `schema.ts`'s `primitiveRegistrySchema` (a zod discriminated
union); only `intent`/`useWhen`/`avoidWhen`/`variants` are hand-authored prose carried forward by component id
across regenerations. `catalog-digest.mjs` then renders the registry into `catalog-digest.md`, the agent-facing
"visual menu" every OTHER node's prompt cites as the sole authoritative primitive inventory (never primitive
source, never a guess). `npm run registry:check` gates drift by regenerating in memory and diffing.
