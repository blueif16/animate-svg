# Why / how it works (the lifecycle, end to end)
The SFX vocabulary + registry (`SFX_REGISTRY`, asset sourcing) live in the external `@studio/sound-kit` package
(`file:../../shared-sound`, same vendored-sibling pattern as narration-kit) — `src/lesson-media/sfx.ts` is a thin
re-export shim preserving the local `./sfx` import path call sites use. What IS local, load-bearing product code
is `LessonSfxLayer.tsx`'s concurrent-audio-tag budget: browsers (and Remotion's render pipeline) cap
simultaneously-decoding `<Html5Audio>` elements, and past that ceiling extra tags silently fail to play with no
error — invisible until a human notices a missing sound in the rendered MP4. `assertConcurrentAudioBudget()`
checks every lesson's declared SFX schedule (authored by w2c into `audio-cues.json`, consumed by the composer
w4a into `sfxEvents`) against the same `MAX_CONCURRENT_AUDIO` ceiling, lesson-agnostically.
