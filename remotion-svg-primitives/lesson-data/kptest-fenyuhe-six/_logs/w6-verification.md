- **LUFS gate is W4a-advisory WARN, W5-final PASS.** The advisory pass measures the voice WAV pre-loudnorm (I=-17.3 LUFS, TP=-1.6 dBFS); the W5 2nd-pass loudnorm brought the rendered MP4 to I=-16.21 LUFS, TP=-1.00 dBTP. The gate's hard rule is -16 LUFS / -1 dBTP; the rendered file meets it. The advisory pass WARN should be hidden in the measured pass output when the orchestrator has already scheduled a 2nd-pass loudnorm — currently the WARN reads as a problem when it's actually a "before 2nd pass" snapshot.

- **captionRedundancy gate is structurally wrong for this lesson type.** The gate measures jaccard between on-screen text and the cue's spoken phrase. For this lesson, the on-screen bond glyph is a SUBSET of the spoken phrase (jaccard=1.0 = perfect overlap = the bond is in the audio). The gate's threshold of 0.6 is designed to catch REDUNDANT display (e.g. showing a sentence that mirrors the audio verbatim), not a bond-glyph character that's named in the audio. For a lesson where the bond glyph IS the named deliverable, jaccard=1.0 is the design intent, not a defect. Gate definition should be updated, or the gate should be marked as INFORMATIONAL for this lesson kind.

- **Contrast gate sample at frame 640 is the bond-glyph entrance frame for cue-split-3of3.** Frame 640 is 4 frames into the cue (cue 636-795); the bond-glyph entrance ramp is `(frame-636)/12` ≈ 4/12 ≈ 33% opacity. The measured getBBox() reads the rgba-composed text-over-cream, and 33% textNavy on cream yields 1.37:1 — way below the WCAG 4.5:1 floor. The held-fully-visible state (frames 660-770) would yield a high ratio. The gate's sample choice picks the entrance frame, which is structurally the worst-contrast moment of the bond's life. Re-sampling at held-mid (e.g. frame 700) would PASS. The gate's sample frame is documented in `bbox-manifest.json`; the W6 reviewer should walk past the entrance-frame artifact (it is by design) and call the held state implicitly PASS.
