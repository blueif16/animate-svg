## ISSUES

## PIPELINE FINDINGS
- The prompt said to read "src/lessons/generated/fenYuHeSilences.ts" as an input, but it does not exist pre-render — it is generated by the render pipeline (Wave 5). The reconcile (Wave 3.5) runs BEFORE render, so the silences module is never present when Wave 3.5 needs it. I generated it via the kit's detect-silences script. The wave contract should explicitly own this: either Wave 3a emits silences alongside the timing module, or Wave 3.5 is documented as the step that runs detect-silences. Currently it is implicit.
- findBoundarySilence is exported from @studio/narration-kit (via ./types barrel) — confirmed, but it is NOT obvious from the index.ts (it re-exports types.ts wholesale). The prompt's instruction to "verify exact export names by reading the package" was necessary and correct.
- The prompt mandated audioBoundary fallback to "the next cue's ASR start" but did not specify the FINAL cue's boundary. I used fenYuHeAlignedDuration (source audio end). This should be stated in the architecture doc.
- "Chained end-to-end from the lead silence" is ambiguous about whether the first cue starts at frame 0 or at the first ASR start (8) or at the lead silence's end. I chose frame 0 (audio start) so the continuous WAV Sequence and the timeline share an origin. The doc should pin this down.
- No prior-lesson timeline was consulted (clean-room respected). The kit types + architecture doc were sufficient to build from scratch.
